## 🐛 MQTT 连接失败排查

### 症状
连接按钮点击后状态显示"错误"或"连接中..."一直卡住

### 可能原因和解决方案

#### ✅ 原因 1: SharkDataSever 未运行

**检查方法**:
\`\`\`bash
# 检查 MQTT 服务是否在运行
netstat -an | findstr :3333
\`\`\`

**解决方案**:
1. 确保 SharkDataSever 正在运行
2. 检查 SharkDataSever 的 MQTT 端口配置
3. 查看 SharkDataSever 的日志

#### ✅ 原因 2: 端口号错误

**解决方案**:
1. 确认 SharkDataSever 的 MQTT 端口（默认 3333）
2. 在应用中修改端口号
3. 点击"连接"按钮重新连接

#### ✅ 原因 3: 防火墙阻止

**检查方法**:
\`\`\`bash
# Windows 防火墙
控制面板 > 系统和安全 > Windows Defender 防火墙
\`\`\`

**解决方案**:
1. 临时关闭防火墙测试
2. 添加应用到防火墙白名单
3. 允许端口 3333 的入站/出站连接

#### ✅ 原因 4: 服务器地址错误

**解决方案**:
1. 如果 SharkDataSever 在本机，使用 `localhost` 或 `127.0.0.1`
2. 如果在其他机器，使用实际 IP 地址
3. 确保网络可达：
   \`\`\`bash
   ping <服务器地址>
   \`\`\`

#### ✅ 原因 5: MQTT 协议版本不匹配

**解决方案**:
1. 确认 SharkDataSever 使用的 MQTT 版本
2. 如果需要，可以在 `mqttService.ts` 中修改：
   \`\`\`typescript
   client = mqtt.connect(url, {
     protocolVersion: 5,  // MQTT 5.0
     // 或
     protocolVersion: 4,  // MQTT 3.1.1
   });
   \`\`\`

## 📊 调试 MQTT 连接

### 方法 1: 查看控制台日志

1. 按 **F12** 打开开发者工具
2. 切换到 **Console** 标签
3. 点击"连接"按钮
4. 查看错误信息：

**常见错误信息**:
| 错误信息 | 原因 | 解决方案 |
|---------|------|----------|
| `ECONNREFUSED` | 服务器未运行 | 启动 SharkDataSever |
| `ETIMEDOUT` | 连接超时 | 检查网络和防火墙 |
| `Proto error` | Protobuf 解析失败 | 已修复，重启应用 |
| `Failed to initialize protobuf` | Proto 加载失败 | 已修复，重启应用 |

### 方法 2: 使用 MQTT 客户端测试

**推荐工具**: MQTT Explorer 或 MQTTX

**测试步骤**:
1. 下载并安装 MQTT Explorer
2. 连接到 `mqtt://localhost:3333`
3. 如果可以连接并看到主题，说明服务器正常
4. 如果不能连接，说明问题在服务器端

### 方法 3: 使用命令行测试

\`\`\`bash
# 使用 mosquitto_sub 测试（需要先安装 mosquitto）
mosquitto_sub -h localhost -p 3333 -t "rm/#" -v

# 如果能看到消息，说明服务器正常
\`\`\`

## 🔌 H.264 视频解码（高级）

如果您的视频流是 H.264 格式，需要额外配置：

### 选项 1: 使用 WebCodecs API（推荐）

WebCodecs 是现代浏览器内置的解码 API，无需额外依赖。

**浏览器要求**: Chrome 94+, Edge 94+

**实现方式**: 修改 `VideoArea.tsx`，添加 WebCodecs 解码器

### 选项 2: 修改服务器输出

**推荐方案**: 让 SharkDataSever 输出 MJPEG 格式

**优点**:
- ✅ 无需客户端解码
- ✅ 性能更好
- ✅ 兼容性更好

## 📝 常见问题 FAQ

### Q1: 为什么 UDP 视频能接收但 MQTT 不能？

**A**: UDP 和 MQTT 是独立的连接：
- UDP 接收不需要握手，只要端口正确就能接收数据
- MQTT 需要完整的 TCP 连接和协议握手
- 检查 MQTT 服务器是否正常运行

### Q2: 视频显示是黑屏或花屏？

**A**: 可能的原因：
1. 视频格式不匹配
2. 分辨率检测错误（原始帧）
3. 数据损坏

**解决方案**:
- 检查"格式"显示
- 尝试使用 MJPEG 格式
- 查看控制台是否有错误

### Q3: FPS 很高但看不到画面？

**A**: 说明数据在接收，但解码有问题：
1. 检查格式显示
2. 如果是 H.264，需要额外解码器
3. 如果是"未知"，数据格式不被识别

### Q4: 应用启动后自动退出？

**A**: 可能是配置文件损坏：
1. 删除配置文件：
   \`\`\`
   %APPDATA%\robomaster-client-native\
   \`\`\`
2. 重新启动应用
